# 热度评分公式

## 综合分数

```
HeatScore = BilibiliSignal(0-45) + CrossPlatformSignal(0-40) + NewsMatchSignal(0-15)
```

总分 0-100 分。最终按此分数降序排列。

---

## BilibiliSignal（0-45 分）

### 自适应百分位制（首选）

当本批次 bilibili 候选数 ≥ 5 时，使用百分位排名替代固定阈值：

1. 将所有 bilibili 候选按播放量降序排列
2. 播放量排名分：
   - Top 20% → +30
   - 20%-40% → +22
   - 40%-60% → +15
   - 60%-80% → +8
   - Bottom 20% → +3
3. 互动量加分（弹幕 + 评论）：
   - 批次内 Top 30% → +8
   - 30%-70% → +5
4. 投币数加分：
   - 批次内 Top 20% → +5
5. 标题/简介时政关键词匹配度：+2

### 固定阈值（回退方案）

当本批次 bilibili 候选数 < 5 时，使用以下固定阈值：

| 条件 | 加分 |
|------|------|
| 播放量 ≥ 100万 | +30 |
| 播放量 30万-100万 | +20 |
| 播放量 5万-30万 | +10 |
| 播放量 1万-5万 | +5 |
| 弹幕 ≥ 5000 或 评论 ≥ 3000 | +8 |
| 投币 ≥ 10000 | +5 |
| 标题时政匹配度 | +2 |

### MCP 字幕/评论加分（如 MCP bilibili 工具可用）

| 条件 | 加分 |
|------|------|
| 视频有可访问字幕 | +3 |
| 热门评论 > 10 条 | +2 |

MCP 工具不可用时静默跳过。

---

## CrossPlatformSignal（0-40 分）

### 正常模式（微博热搜数据可用）

| 条件 | 加分 |
|------|------|
| 话题出现在 bilibili 热搜 Top5 | +20 |
| 话题出现在 bilibili 热搜 Top20 | +12 |
| 话题出现在微博热搜 Top10（hot_value ≥ 100万） | +15 |
| 话题出现在微博热搜 Top50 | +8 |

### 降级模式（仅 bilibili 热搜，无微博数据）

CrossPlatformSignal 上限降为 **20 分**。

| 条件 | 加分 |
|------|------|
| 话题出现在 bilibili 热搜 Top5 | +12 |
| 话题出现在 bilibili 热搜 Top20 | +8 |

### 话题匹配规则

提取 bilibili 视频标题中的核心关键词（去除标点、保留人名/事件名），
与微博热搜 title 做模糊匹配（包含关系即算匹配）。

---

## NewsMatchSignal（0-15 分）

传统媒体报道覆盖度：

| 条件 | 加分 |
|------|------|
| 3+ 传统媒体报道同一事件 | +12 |
| 2 家报道 | +8 |
| 1 家报道 | +4 |

> 注意：去重（`tools/dedup.py`）后此分数自动生效，不再单独计算。

---

## 来源多样性加分（去重阶段自动计算）

去重后，同一事件跨平台覆盖的额外加分：

| 不同来源数 | 加分 |
|-----------|------|
| 4+ | +9 |
| 3 | +6 |
| 2 | +3 |
| 1 | +0 |

此加分在 `tools/dedup.py` 中自动计算，合并到最终 HeatScore。

---

## 偏好话题加分

如果 `config.yaml` 中配置了 `favorite_topics`：
- 匹配到偏好话题的条目在最终 HeatScore 中 **+5**

此加分在 `tools/classify.py` 标注话题标签后应用。

---

## 排名规则

1. **Bilibili 时政视频**：按 HeatScore 降序，取前 N 条（目标占 40%）
2. **热搜新闻**：从热搜 + 传统媒体中筛选，按 HeatScore 排序
3. **传统新闻**：按 HeatScore 排序，排除已被 bilibili/热搜覆盖的
4. **最终比例**：bilibili ~40%，传统+热搜 ~60%
5. **总量控制**：10-15 条
